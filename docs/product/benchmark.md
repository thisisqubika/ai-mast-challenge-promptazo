# Fan Fest — Benchmark

Source: Benchmark FigJam → Page 1 (competitor research). What each reference does
well and what Fan Fest should borrow.

| Platform | Strength | Borrow for Fan Fest |
|---|---|---|
| **Eventbrite** | Gold standard for event creation + ticketing; clean create→publish flow | Minimal-step event creation and RSVP flow |
| **Fever** | Location-aware feed, visual-first event cards, urgency mechanics ("selling fast") | The look/feel of the **home discovery feed** — discovery as an experience, not a list |
| **Meetup** | Community/group model; **attendee-list visibility** | Show who else is going — social proof drives RSVP conversion |
| **Resident Advisor** | Geolocation discovery, **Maps/Uber navigation** integration, deep event pages | Maps + navigation to the venue; rich event detail pages |
| **Apple Invites / Facebook Events** | Invite-by-link, **shared photo wall**, guest-list social proof | Frictionless invite + the **Hype Wall** shared-album model |

## Design takeaways

- **Discovery feel** → Fever. Visual, vibe-first cards, location-aware.
- **Conversion driver** → Meetup. Visible attendee list = "my friends are going".
- **Navigation** → Resident Advisor. One-tap directions to the venue (Google Maps).
- **Shared memory** → Apple Invites / FB Events. Attendees co-build the photo wall.

## Differentiator

None of the references offer anything **after the final whistle**. Fan Fest's
AI **Recap** (highlights, result, prediction outcomes, photo carousel,
next-event suggestions) is the feature no competitor has — and the demo's hook.
